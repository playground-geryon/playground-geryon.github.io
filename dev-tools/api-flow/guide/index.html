API 등록 및 플로우 기반 테스트

소개

API를 등록하고 플로우로 호출합니다.

가이드
공통 설정
공통 헤더/전역 변수를 관리합니다.
  • [공통 설정] 클릭
  • 헤더 또는 변수 탭에서 추가/수정
결과: 공통 값이 전체 노드에 재사용됩니다.
공통 헤더 사용
노드별로 공통 헤더 적용 여부를 설정합니다.
  • 노드 상세에서 [공통 헤더 사용] 스위치 ON/OFF
결과: 선택한 노드에 공통 헤더가 포함됩니다.
API 추가
노드를 추가하고 목록에서 선택합니다.
  • 상단 [추가] 클릭
  • 좌측 리스트에서 노드 선택
  • 노드명 추가
  • 노드 설명 추가
  • URL/메서드 입력
  • 요청 항목 설정
결과: 선택한 노드의 상세 설정이 우측에 표시됩니다.
플로우 실행
등록된 노드를 실행합니다.
  • 플로우 연결
  • 실행 클릭
결과: 응답이 우측에 표시됩니다.
전달 변수 설정
응답에서 값을 추출해 다음 노드로 전달합니다.
  • [다음 노드 자동 실행] 활성화
  • [전달 변수 설정]에서 JSONPath 입력
결과: 추출된 값이 {{변수명}} 형태로 사용됩니다.
저장 [중요]
현재 플로우 상태를 저장합니다. 저장을 안하면 추가된 노드 정보가 초기화 됩니다.
  • 상단 [저장] 클릭
결과: 변경 사항이 저장되고 저장 표시가 사라집니다.
요청 탭
요청 구성 요소를 탭별로 설정합니다.
  • 요청 헤더 / Query / Path / Request Value / 요청 로그 탭 전환
결과: 요청 데이터가 분리된 탭에서 관리됩니다.
응답 탭
응답 본문/헤더를 탭별로 확인합니다.
  • 응답 본문 / 응답 헤더 탭 전환
  • 필요 시 [실행] 클릭
결과: 응답 결과와 헤더를 각각 확인할 수 있습니다.
